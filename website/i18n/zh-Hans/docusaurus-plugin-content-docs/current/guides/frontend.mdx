
# 前端

## 脚本注入

当Wail服务于您的 `索引时。 tml`, 默认, 它将注入2个脚本条目到 `<body>` 标签以加载 `/wails/ipc。 s` 和 `/wails/runtime.js`. 这些文件分别安装绑定和运行时间。

下面的代码显示默认注入的位置：

```html
<html>
<head>
  <title>注入示例</title>
  <link rel="stylesheet" href="/main.css">
  <！ -     <script src="/wails/ipc.js"></script> -->
  <！ -     <script src="/wails/runtime.js"></script> -->
</head>

<body data-wails-drag>
  <div class="logo"></div></div>
  <div class="result" id="result">请输入您的名字在 👇</div>
  <div class="input-box" id="input" data-wails-no-drag>
    <input class="input" id="name" type="text" autocomplete="off">
    <button class="btn" onclick="greet()">Greet</button>
  </div>

  <script src="/main.js"></script>
</body>

</html>
```

### 覆盖默认脚本注入值

为了给开发者提供更多的灵活性，有一个元标签可以用于自定义此行为：

```html
  <meta name="wails-options" content="[options]">
```

备选办法如下：

| 值                | 描述                         |
| ---------------- | -------------------------- |
| noautointruntime | 禁用自动注入 `/wails/runtime.js` |
| noautoinjectipc  | 禁用自动注入 `/wails/ipc.js`     |
| noautoinject     | 禁用所有自动插入脚本                 |

可以使用多个选项，如果它们是逗号分隔的。

此代码完全有效，并且与自动注入版本操作相同：

```html
<html>

<head>
  <title>注入示例</title>
  <meta name="wails-options" content="noautoinject">
  <link rel="stylesheet" href="/main.css">

</head></head>

<body data-wails-drag>
  <div class="logo"></div>
  <div class="result" id="result">请填写您的名字在 👇</div>
  <div class="input-box" id="input" data-wails-no-drag>
    <input class="input" id="name" type="text" autocomplete="off">
    <button class="btn" onclick="greet()">Greet</button>
  </div>

  <script src="/wails/ipc.js"></script>
  <script src="/wails/runtime.js"></script>
  <script src="/main.js"></script>
</body>

</html>
```
