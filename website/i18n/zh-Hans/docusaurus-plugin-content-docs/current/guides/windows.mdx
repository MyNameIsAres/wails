
# 窗口

此页面有与开发Windows应用相关的杂项指南。

## 处理 WebView2 运行时依赖关系

Windows构建的Wail应用程序在Microsoft [WebView2 Runtime](https://developer.microsoft.com/en-us/microsoft-edge/webview2/) 上有运行时间。 Windows 11默认情况下会安装这个设备，但是有些机器不会安装。 邮寄为处理这种依赖提供了一种简单的方法。

在构建时使用 `-webview2` 标志， 您可以决定当没有检测到合适的运行时间时您的应用程序将做些什么(包括如果安装的运行时间太老)。 这四个备选方案是：

1. 下载
2. 嵌入
3. 浏览器
4. 错误

### 下载

此选项将提示用户没有找到合适的运行时间，然后提议从 Microsoft 的 WebView2 网站下载和运行官方 引导器。 如果用户继续，官方引导程序将被下载并运行。

### 嵌入

此选项在应用程序中包含官方引导器。 如果没有找到合适的运行时间， 应用程序将提供运行引导器。 这会将~150k添加到二进制文件大小。

### 浏览器

此选项将提示用户没有找到合适的运行时间，然后提议打开浏览器到官方 WebView2页面，在那里可以下载和安装bootstraper。 然后应用程序将退出，将安装 留给用户。

### 错误

如果没有找到合适的运行时间，用户将会出现错误，没有采取进一步的行动。

## 固定版本运行时间

处理webview2依赖关系的另一种方式是将它送到自己。 您可以下载 [固定版本运行时](https://developer.microsoft.com/ru-ru/microsoft-edge/webview2/#download-section) 和捆绑或者用您的应用程序下载它。

另外，您应该在 `窗口中指定固定版本的 webview2 运行时间的路径。启动唤醒时选项` 结构。

```go
    运行(&options.App@un.org
        Windows： &winds.Options@un.org
            WebviewBrowserPath：""，
        }，
})
```

注意：当 `WebviewBrowserPath` 被指定时 `错误` 策略将在最小需要的版本不匹配或无效路径到运行时间时强制执行。
