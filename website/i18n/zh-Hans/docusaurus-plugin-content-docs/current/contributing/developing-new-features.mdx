# 开发新功能

我们总是热衷于在Wail中添加功能并扩展项目可以做些什么。 增加新功能的过程如下：

- 选择 "TODO" 标签的增强服务单。 最好从当前的 [积压](https://github.com/orgs/wailsapp/projects/1/views/1) 中选择一个，但是选择是你的。
- 在开发之前，请检查工单是否包含以下信息：
- 增强工作的目的
- 什么是无法增强的
- 升级目标的平台 (大多数功能应该是跨平台的，除非有非常具体的理由)
- 如果工单没有包含此信息，请随时向打开工单的 个人索取信息。 有时创建占位符工单并需要更多详细信息
- 对工单的评论，您希望开发该功能
- 复制仓库并创建一个分支，格式为 `feature/<ticket_number>_<ticket_title>`
- 新功能经常需要文档，请确保您也添加或更新文档作为 更改的一部分。
- 一旦特性准备好测试，创建一个 PR草案。 请确认 PR 描述有测试场景和 带有检查标记的测试案例， 使其他人能够知道仍然需要测试的内容。
- 一旦完成所有测试，请从草稿中更新PR 状态并留下一条消息。

:::note
没有任何东西阻止您打开工单并自己工作。 但请注意，所有
增强请求都经过了审核。 并非所有的想法都会被选中，所以最好在工单上先进行讨论
。
:::

:::警告
在没有相应服务单的情况下打开的任何PR都可能被拒绝。
:::
