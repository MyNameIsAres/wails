# 修复错误

修复bug的过程如下：

- 检查当前的 [待办事宜](https://github.com/orgs/wailsapp/projects/1/views/1) 并选择一个 bug 以修复
- 在开发之前，请检查工单是否包含以下信息：
- A. 问题的范围，包括受影响的平台
- 要复制的步骤。 有时出现了不属于邮寄问题的故障，报告者有责任 证明它是一个用最小的可复制示例的遗迹问题
- `视觉医生的输出`
- 如果工单没有包含此信息，请随时向打开工单的 个人索取信息。
- 对工单的评论，您希望开发一个修复
- 复制仓库并创建一个分支，格式为 `bugfix/<ticket_number>_<ticket_title>`
- 一旦修复准备好进行测试，创建一个 PR草案。 请确认 PR 描述有测试场景和 带有检查标记的测试案例， 使其他人能够知道仍然需要测试的内容。
- 一旦完成所有测试，请从草稿中更新PR 状态并留下一条消息。

:::note
没有任何东西阻止您打开工单并自己工作。 但请注意，应该讨论所有
bug修复，因为该方法可能会产生意外的副作用。
:::

:::警告
在没有相应服务单的情况下打开的任何PR都可能被拒绝。
:::
