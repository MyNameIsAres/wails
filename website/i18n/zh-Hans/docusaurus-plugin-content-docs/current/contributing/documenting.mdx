# 正在文档

该网站也是该项目的主要文件网站。 有时候这会使 过时，需要一些小的调整。 有些文档也没有按照最佳标准写入 。 开发文档是很困难的，因此我们非常感谢 的任何贡献。 没有文档的功能是未完成的 个项目，它是 *个重要的* 作为代码。

我们通常不创建更新文档的工单，所以如果有文本， 您认为应当更新或改写，然后可以随时为此提交PR 。 此网站 位于主仓库的 `网站` 目录下。 我们使用 [Docusaurus](https://docusaurus.io/) 来创建 站点，所以有大量的现有文档和教程可以开始。

为了建立地方文件编制环境，采取以下行动：

- [Install npm](https://docs.npmjs.com/cli/v8/configuring-npm/install)
- `cd 网站`
- `npm install`
- `npm 运行开始`

在所有安装完毕并运行后，您应该看到网站 [`http://localhost:3000`](http://localhost:3000) 对站点文本的任何更改将立即反映在浏览器中。

## Versioning

我们使用一个版本系统，在这个系统中，我们拥有"最新"文档AKA"下一个版本"，该版本 包含自上次发布以来发生的所有更改。 我们还保留上次发布的 文档以及之前的版本。

通常没有理由更新发布的文档，所以我们通常不在 `versioned_docs` 或 `versioned_sidars` 目录中更新 文档。

"下一版"文档主要是在 `网站/文档` 中，一些"版本独立"文档 在 `src/pages` 中。 任何更新都应该在 `网站/文档` 目录中进行。
