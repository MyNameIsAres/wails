# Hello World

本教程的目的是让你使用最基本的 软件来运行和使用Wail。 您将能够：

  - 创建一个新的Wail应用程序
  - 构建应用程序
  - 运行应用程序

::note 此教程使用 Windows 作为目标平台。 输出将根据您的操作系统略微变化 。 :::

## 创建一个新的Wail应用程序

要使用默认的原版JS模板创建一个新的Wail应用程序， 您需要运行以下命令：

```bash
行内-n Helloworld
```

您应该看到类似于以下内容的东西：

```
Wails CLI v2.0

初始化项目 'helloworld'
--------------------------------------------------------

项目名称: helloworld
项目目录: C:\Users\leaan\tutorial\helloworld
项目模板: vanilla
模板支持: https://wails. o

在232ms中初始化项目“helloworld”。
```

这将在当前目录中创建一个名为 `helloworld` 的新目录。 在这个目录中，您会找到一些文件：

```
build/ - 包含构建文件 + 编译的应用程序
前端/ - 包含前端文件
应用。 o -包含应用程序代码
Main。 o - 具有应用程序配置的主要程序
wails.json - 项目配置文件
过了。 od - 模块文件
过了。 um - 模块校验和文件
```

## 构建应用程序

要构建应用程序，请更改为新的 `helloword` 项目目录并运行以下命令：

```bash
构建Wails
```

你应该看到如下东西：

```
Wails CLI v2.0

应用类型：桌面
平台：windows/amd64
编译器：C:\Users\leaan\go\go1。 8.3\bin\go。 xe
构建模式：生产
跳过前端： false
压缩： false
Package： true
Clean Building Dir： false
LDFlags： ""
标签： […]
种族检测器： false

Building target： windows/amd64
--------------------------------------
  - 安装前端依赖： 完成。
  - 编译前端：完成。
  - 生成捆绑资产：完成。
  - 编译应用程序：完成。
在 10.616 秒内生成'C:\Users\leaan\tutorial\helloworld\build\bin\helloworld.exe'
```

这已经编译了应用程序并保存在 `build/bin` 目录中。

## 运行应用程序

如果我们在 Windows Explorer 中查看 `build/bin` 目录，我们应该看到我们的项目二进制：

    <img src="/img/helloworld-app-icon.png" width="134px"/>

<br/>

我们可以通过简单的双击 `helloworld.exe` 文件来运行它。

在 Mac 上，Wails 生成一个 `helloworld.app` 文件，可以通过双击来运行。

在 Linux 上，您可以使用 `/helloword` 从 `build/bin` 目录运行应用程序。
