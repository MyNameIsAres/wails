# 工程配置

项目配置位于项目目录中的 `wails.json` 文件。 配置结构为：

```json
{
    "name": "[The project name]",
    "assetdir": "[Relative path to the directory containing the compiled assets, this is normally inferred and could be left empty]",
    "reloaddirs": "[Additional directories to trigger reloads (comma separated), this is only used for some advanced asset configurations]",
    "frontend:install": "[The command to install node dependencies, run in the frontend directory - often `npm install`]",
    "frontend:build": "[The command to build the assets, run in the frontend directory - often `npm run build`]",
    "frontend:dev": "[This command is the dev equivalent of frontend:build. 如果未指明后退到前端：构建]”，
    "frontend:dev:watcher"：“[此命令是在`wails dev`单独的进程中运行的。 对第三方观察者或启动 3d 派对服务器非常有用。]”，
    "frontend:dev:serverUrl": "[URL到第三方开发服务器以用于服务资产, EG Vite。 如果设置为“自动”，那么devServerUrl将被从Vite输出推断出来。]”
    "wailsjsdir"：“[将创建自动生成的 JS 模块的相对路径]”，
    "版本": "[项目配置版本]",
    "outputfilename": "[二进制文件名称]",
    "debounceMS": 100, // 当dev 服务器检测到资产变化时默认等待重新加载的时间
    "devServer": "[地址绑定断裂器. 默认值：localhost:34115]",
    "appargs": "[在 dev 模式下以shell 风格传递给应用程序的参数]",
    "runNonativeBuildHooks": false, // 定义构建钩子是否应该运行，它们是为主机OS以外的操作系统定义的。
    "PreBuilding Hooks": 让您能
        "GOOS/GOARCH": "[在构建指定的 GOOS/GOARCH之前将执行的命令: ${platform} 被替换为“GOOS/GOARCH”。 “GOOS/GOARCH”钩子是在“GOOS/*”和“*/*”钩子之前执行的。 ",
        "GOOS/*": "[在构建指定的GOOS之前将执行的命令: ${platform} 被替换为“GOOS/GOARCH”。 “GOOS/*”钩子是在"*/*"钩子之前执行的。 ",
        "*/*": "[每个构建前将执行的命令: ${platform} 被替换为 "GOOS/GOARCH"。]"
    },
    "postBuilding Hooks": ~
        "GOOS/GOARCH": "[在构建指定的 GOOS/GOARCH后将执行的命令: ${platform} 被"GOOS/GOARCH" 和 ${bin} 替换为被编译的二进制文件。 “GOOS/GOARCH”钩子是在“GOOS/*”和“*/*”钩子之前执行的。 ",
        "GOOS/*": "[在构建指定的 GOOS之后将执行的命令: ${platform} 被替换为 "GOOS/GOARCH" 和 ${bin} 被编译的二进制文件路径。 “GOOS/*”钩子是在"*/*"钩子之前执行的。 ",
        "*/*": "[每个构建后将执行的命令: ${platform} 被替换为 "GOOS/GOARCH" 和 ${bin} 被编译的二进制文件路径。]"
    },
    "information": 让您// Data 用于填充清单和版本信息。
        “公司名称”：“[公司名称]。 默认：[项目名称]]”，
        "productName": "[产品名称"。 默认值：[项目名称]]”，
        "productVersion": "[产品版本号"。 默认：'1.0.0']”，
        "版权": "[产品的版权。 默认：'版权......']”，
        "评论": "[应用程序的简短评论。 默认: '使用 Wails (https://wails.app)']"
    },
    "nsisType": "['multiple': 每个建筑一个安装程序。 '单一'：正在构建的所有建筑物的单一通用安装程序。 默认值: 'multiple']"
}
```

此文件由Wails CLI在运行 `wails build` 或 `wails dev` 时阅读。

`assetdir`, `reloadaddurs`, `wailsjsdir`, `debounceMS`, , `devserver` 和 `frontenddevserverurl` 标记在 `wails build/dev` 将更新项目配置 从而成为以后运行的默认。
