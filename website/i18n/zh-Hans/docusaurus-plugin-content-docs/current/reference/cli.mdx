# CLI

Wails CLI 有一些命令用于管理您的项目。 所有命令都以以下方式运行：

`wails <command> <flags>`

## init

`嵌入` 用于生成项目。

| 标志               | 描述                                       |  默认设置   |
|:---------------- |:---------------------------------------- |:-------:|
| -n "项目名称"        | 项目名称 **Mandatory**.                      |         |
| -d "project dir" | 要创建的项目目录                                 |  项目名称   |
| -g               | 初始化git 仓库                                |         |
| -l               | 列出可用的项目模板                                |         |
| -q               | 禁止输出到控制台                                 |         |
| -t "模板名称"        | 要使用的项目模板。 这可以是默认模板或在 githb 托管的远程模板的 URL。 | vanilla |
| -ide             | 生成 IDE 工程文件                              |         |
| -f               | 强制构建应用程序                                 |  false  |

示例： `wails init -n test-d mytestproject -g -ide vscode -q`

这将在"mytestproject"目录中生成一个名为 "test" 的项目，初始化git， 生成vscode 项目文件，并静默地这样做。

关于使用 IDEs Wails 的更多信息可以在这里找到 [](../guides/ides.mdx)。

### 远程模板

远程模板 (在 GitHub 上托管) 得到支持，可以通过使用模板的项目 URL 安装。

示例： `wails init -n test-t https://github.com/leaanthony/testtemplate[@v1.0.0]`

社区维护模板列表可以在这里找到 [](../community/templates.mdx)

:::警告注意事项

  **Wails项目不维护，对第三方模板不负责任或责任！**

  如果您对模板不确定，请检查 `package.json` 和 `wails.json` 查看哪些脚本正在运行和哪些软件包已安装。

:::

## 构建中

`wails build` 被用于编译你的项目到准备好生产的二进制文件。

| 标志                 | 描述                                                                                                                                      | 默认设置                                                                                                  |
|:------------------ |:--------------------------------------------------------------------------------------------------------------------------------------- |:----------------------------------------------------------------------------------------------------- |
| -平台                | Build for the given (comma delimited) [platforms](../reference/cli.mdx#platforms) eg. `windows/arm64`. 注意，如果您没有给出架构， `运行时间。GAARH` 已被使用。 | 平台 = `GOOS` 环境变量，如果另外给出 `运行时间。GOOS`。<br/>arch = `GOARCH` envrionment 变量如果给出其他 `runtime.GOARCH`. |
| -清理                | 清除 `生成/bin` 目录                                                                                                                          |                                                                                                       |
| -编译器"编译器"          | 使用不同的编译器来构建，例如 go1.15bet1                                                                                                               | 转至                                                                                                    |
| -ldflags "flags"   | 添加要传递到编译器的 ldflags                                                                                                                      |                                                                                                       |
| -不拍照               | 不要打包应用程序                                                                                                                                |                                                                                                       |
| -o 文件名             | 输出文件名                                                                                                                                   |                                                                                                       |
| - 秒                | 跳过建立前端                                                                                                                                  | false                                                                                                 |
| -f                 | 强制构建应用程序                                                                                                                                | false                                                                                                 |
| -tags "extra tags" | 构建标签传递给编译器 (引用和空间分隔)                                                                                                                    |                                                                                                       |
| -up                | 使用 "ux" 压缩最后二进制文件                                                                                                                       |                                                                                                       |
| -upxflags          | 要传递到上方的标记                                                                                                                               |                                                                                                       |
| -v int             | 详细级别 (0-静音, 1 - 默认, 2 - 详细)                                                                                                             | 1                                                                                                     |
| -webview2          | WebView2 安装程序策略: download,embed,browser,错误                                                                                              | 下载                                                                                                    |
| -u                 | 更新你的项目的 `go.mod` 来使用与 CLI 相同的版本Wails                                                                                                    |                                                                                                       |
| -debug             | 在应用程序中获取调试信息。 允许在应用程序窗口中使用 devtools                                                                                                     | false                                                                                                 |
| -trimpath          | 从生成的可执行文件中删除所有文件系统路径。                                                                                                                   | false                                                                                                 |
| -竞技场               | 使用 Go的种族探测器建设                                                                                                                           | false                                                                                                 |
| -windsconsole      | 为Windows构建保留控制台窗口                                                                                                                       | false                                                                                                 |

`webview2` 标志的详细描述，请参阅 [Windows](../guides/windows.mdx) 指南。

如果您喜欢使用标准的去配刀构建，请参阅 [手册编译](../guides/manual-builds.mdx) 指南。

示例：

`wails build-clean-o myproject.exe`

:::info UPX 在 Apple Silicon

  There are [issues](https://github.com/upx/upx/issues/446) with using UPX with Apple Silicon.

:::

:::info UPX 在 Windows 上

  一些抗病毒销售商错误地将 `升到` 压缩二进制病毒，见 [问题](https://github.com/upx/upx/issues/437)。

:::

| 窗户/arm64      | Windows 10/11 ARM64     |

| linux/arm64   | Linux ARM64             |

## dev

`wails dev` 被用来运行您的应用程序在 "live development" 模式。 这意味着：

   - 应用程序的 `go.mod` 将被更新以使用与 CLI 相同版本的Wails
   - 应用程序被自动编译和运行
   - 一个监视器已经启动，如果检测到您的数据文件的更改，将会触发你的开发应用的重建
   - Web 服务器已在 `http://localhost:34115` 上启动，后者通过 http 为您的应用程序(不仅仅是前端)服务。 这允许您使用您最喜欢的浏览器开发扩展
   - 所有应用程序资源都从磁盘加载。 如果更改，应用程序将自动重新加载(而不是重建)。 所有已连接的浏览器也将重新加载
   - 生成了一个提供以下服务的JS模块：
     - 自动生成的 JSDoc，提供代码提示的 Javascript 转手方法
     - 您的 Go 结构的 TypeScript 版本，可以构造并传递到您的去方法
   - 生成了第二个JS模块，为运行时提供一个包装器 + TS 声明

| 标志                           | 描述                                                                                                                  | 默认设置              |
|:---------------------------- |:------------------------------------------------------------------------------------------------------------------- |:----------------- |
| -assetdir "./path/to/assets" | 从给定目录服务资产，而不是使用提供的资产 FS                                                                                             | `wails.json` 中的值  |
| -浏览器                         | 启动时打开浏览器到 `http://localhost:34115`                                                                                  |                   |
| -编译器"编译器"                    | 使用不同的编译器来构建，例如 go1.15bet1                                                                                           | 转至                |
| -e                           | 触发重建的扩展 (逗号分隔)                                                                                                      | 转至                |
| -reloaddirs                  | 触发重新加载的附加目录(逗号分隔)                                                                                                   | `wails.json` 中的值  |
| -ldflags "flags"             | 添加要传递到编译器的 ldflags                                                                                                  |                   |
| -tags "extra tags"           | 构建标签传递给编译器 (引用和空间分隔)                                                                                                |                   |
| -loglevel "loglevel"         | 要使用的日志级别 - Trace, Debug, Info, 警告, 错误                                                                               | Debug             |
| -noreload                    | 当资产改变时禁用自动重新加载                                                                                                      |                   |
| -nogen                       | 禁用生成模块                                                                                                              |                   |
| -v                           | 详细程度 (0-静音, 1 - 标准, 2 - 详细)                                                                                         | 1                 |
| -wailsjsdir                  | 生成的Wails JS 模块的目录                                                                                                   | `wails.json` 中的值  |
| -debounce                    | 检测到资产变更后等待重新加载的时间                                                                                                   | 100 (毫秒)          |
| -devserver "host:port"       | 将Wails dev 服务器绑定到的地址                                                                                                | "localhost:34115" |
| -frontenddevserverurl "url"  | 使用第三方开发服务器URL为资产服务，EG Vite                                                                                          | ""                |
| -appargs "args"              | 使用 shell 风格传递到应用程序的参数                                                                                               |                   |
| -保存                          | 保存给给定的 `assetdir`, `重新加载者`, `wailsjsdir`, `反弹`, `devserver` 和 `frontenddevserverurl` 标记在 `wails。 son` 将成为随后被调用的默认值。 |                   |
| -竞技场                         | 使用 Go的种族探测器建设                                                                                                       | false             |
| - 秒                          | 跳过建立前端                                                                                                              | false             |

示例：

`wails dev -assetdir ./frontend/dist -wailsjsdir ./frontend/src -浏览器`

此命令将执行以下操作：

  - 构建应用程序并运行它(更多详细信息 [在这里](../guides/manual-builds.mdx)
  - 在 `./frontend/src 中生成Wails JS 模块`
  - 监视对 `中的文件的更新。/frontend/dist` 并重新加载任何更改
  - 打开浏览器并连接到应用程序

There is more information on using this feature with existing framework scripts [here](../guides/application-development.mdx#live-reloading).

## 生成

### 模版

Wail使用模板生成项目。 `wails 生成模板` 命令帮助手写手写一个模板，以便 它可以用于生成项目。

| 标志      | 描述             |
|:------- |:-------------- |
| -名称     | 模板名称(随机)       |
| -前端"路径" | 要在模板中使用的前端项目路径 |

关于创建模板的详细信息，请参阅 [模板指南](../guides/templates.mdx)。

## 更新

`等待更新` 将更新Wails CLI 版本。

| 标志            | 描述          |
|:------------- |:----------- |
| -pre          | 更新到最新的预发布版本 |
| -版本 "version" | 安装指定版本的 CLI |


## 版本

`wails 版本` 将只是输出当前的 CLI 版本。
