# 事件

## 概览

Wail运行时提供了一个统一的事件系统，在这个系统中，事件可以被传送或接收到去或Javascript。 可选择通过事件传递数据。 监听器将收到本地数据类型中的数据。

### 事件

去签署: `EventsOn(ctx context.Context, eventname 字符串, callback functionalData ...interface{}))`

JS 签署： `EventsOn(eventname 字符串，回调函数(可选Data?：任何))`

这个方法设置给定事件名称的监听器。 当类型为 `事件名称` 的事件 [已经发出](#EventsEmit), 会触发回调。 任何与发出的事件一起发送的额外数据都将传递给回调。

### 事件关闭

去签署: `EventsOff(ctx context.Context, eventname 字符串)`

JS 签署： `EventsOff(eventname 字符串)`

这个方法取消了给定事件名称的监听器。

### EventsOnce

去签署: `EventsOnce(ctx context.Context, eventname 字符串, callback functionalData ...interface{}))`

JS 签署： `EventsOnce(eventname 字符串，回调函数(可选的数据？：任何))`

此方法为给定事件名称设置一个监听器，但只触发一次。

### EventsonMultiple

去签署: `EventsOnMultiple(ctx context. Context, eventname 字符串, callback functionalData ...interface{}), 计数器`

JS 签署： `EventsOnMultiple(eventname 字符串，回调函数(可选的数据？：任何)，计数器)`

这个方法为给定事件名称设置了一个监听器，但只会触发最多 `计数器` 次。

### 事件

去签署: `EventsEmit(ctx context.Context, eventname 字符串, optionalData ...interface{})`

JS 签署： `EventsEmit(ctx context, optionalData functionalData?: any))`

这个方法发布给定的事件。 可选的数据与事件一起传递。 这将触发任何事件监听器。
